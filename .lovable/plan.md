

# 고정 패턴 시스템 근본 문제 수정

## 현재 문제 원인

스크린샷의 문장은 `Crises of this ilk are precisely the kind that can easily engulf you...`인데, 출력에는 `관계대명사 what이 이끄는 명사절(What~important)`이 나옴. 이 문장에 "what"은 없음.

원인은 2가지가 결합됨:

### 원인 1: Grammar Track이 모든 문법 태그 패턴을 무조건 프롬프트에 주입
- `GRAMMAR_TAGS`에 해당하면 문장과 단어 겹침 여부를 전혀 안 봄 (line 824-826)
- DB에 저장된 `관계대명사` 고정 패턴이 "what이 이끄는 명사절(What~important)" 같은 **특정 문장 전용 예시**일 경우, "that"이 있는 문장에도 그대로 주입됨
- AI가 이걸 "필수 적용 규칙"으로 받고 그대로 복사

### 원인 2: 후처리가 패턴을 문장에 맞게 변환하지 못함
- `materializePinnedPattern`에서 `___`가 없는 패턴은 **원문 그대로 반환** (line 465-466)
- 즉 "what이 이끄는 명사절(What~important)"이 그대로 출력됨
- 설령 AI가 "that이 이끄는 관계절(that~them)"로 올바르게 생성해도, 후처리에서 고정 패턴으로 덮어씌워짐

### 핵심 모순
고정 패턴의 목적은 **말투와 양식(스타일)**을 강제하는 것인데, 현재 구조는 **내용(특정 단어/구문)**까지 강제함. "what~important"는 스타일이 아니라 특정 문장의 내용임.

## 수정 방향

### 1. 고정 패턴을 "스타일 템플릿"으로만 사용 — 내용은 AI가 채움

후처리 `materializePinnedPattern`의 동작을 변경:
- `___`가 없는 패턴도, AI 출력의 영어 단어/구문을 추출해서 패턴의 영어 부분을 교체
- 예: 고정 패턴 `관계대명사 what이 이끄는 명사절(What~important)이 문장의 주어 역할`
- AI 출력: `관계대명사 that이 이끄는 관계절(that~them)이 선행사 kind를 수식`
- 결과: **패턴의 한국어 구조/말투는 유지**하되, **영어 단어는 AI 출력에서 가져옴**

### 2. 프롬프트 주입 방식 변경 — "내용 복사"가 아닌 "스타일 참고"

현재:
```
[필수 적용 규칙] — 아래 패턴의 구조·어휘·종결 방식을 정확히 복제하라
```
변경:
```
[필수 적용 규칙] — 아래 패턴의 설명 구조·말투·종결 방식을 따르되, 
영어 단어와 구문 범위는 반드시 현재 문장의 실제 내용으로 교체하라.
패턴에 있는 영어 단어(예: what, important, built 등)를 그대로 쓰지 말 것.
현재 문장에 없는 단어가 출력에 포함되면 오류다.
```

### 3. Grammar Track 필터에 최소 안전장치 추가

태그 매칭만으로 포함하되, 패턴 내 영어 구문이 문장에 전혀 없으면 프롬프트에는 포함하고 `byTag` 후처리에서는 제외:
- 프롬프트: 스타일 참고용으로 포함 (AI가 말투를 따라갈 수 있도록)
- `byTag` 후처리: 패턴의 핵심 문법 키워드(관계대명사, 분사, 수동태 등 한국어 태그)가 AI 출력에 있을 때만 교체 실행
- 이렇게 하면 AI가 "that"을 정확히 분석한 후, 후처리에서 "what~important"로 덮어씌우는 것을 방지

## 수정 파일

| 파일 | 변경 |
|------|------|
| `supabase/functions/grammar/index.ts` | `materializePinnedPattern` — `___` 없는 패턴도 AI 출력에서 영어 단어 추출하여 교체. 프롬프트 문구 변경. `byTag` 후처리 시 최소 검증 추가 |
| `supabase/functions/grammar-chat/index.ts` | 동일한 `materializePinnedPattern` 로직 동기화 |

## 기대 결과

- 고정 패턴의 **말투·양식·종결 방식**은 강제 적용됨
- 고정 패턴의 **영어 단어·구문 범위**는 현재 문장에 맞게 자동 교체됨
- "what~important"가 "that" 문장에 나타나는 현상 완전 제거
- "rather than" 같은 표현형 오염도 기존 Phrase Track 필터로 계속 차단
