

# Passage Logic 길이 강제 — 프롬프트 강화 + 자동 재요청

## 수정 파일
`supabase/functions/analyze-preview/index.ts` (1개)

## 변경 1 — 시스템 프롬프트 최상단에 길이 규칙 박스 추가

프롬프트 맨 앞(페르소나 문장 직전)에 삽입:

```
[CRITICAL LENGTH RULE — 최우선 규칙]
summary의 각 줄(①②③④)은 반드시 한국어 45~58자 (공백·번호·구두점 포함).
- 40자 미만 = 무효. 출력 금지.
- 출력 직전 각 줄 글자수를 직접 세어 검증할 것.
- 짧으면 주체/메커니즘/구체 개념을 1개 추가해 늘릴 것.
- "간결하게" 쓰려는 본능을 누르고 정보를 채워 넣을 것.
이 규칙은 다른 모든 스타일 규칙보다 우선한다.
```

## 변경 2 — Few-shot Bad 예시 교체

기존의 23자짜리 Bad 예시는 거꾸로 모델이 따라할 위험이 있음 → 실제 짧게 나오는 패턴(28~32자)으로 교체:

```
Bad (40자 미만 — 정보 부족, 절대 금지):
① 외부적 보상이 단기 동기 부여에 미치는 한계  ← 24자, 무효
② 인센티브와 처벌이 초래하는 비용과 스트레스  ← 22자, 무효
→ 위 길이는 모두 무효. 주체/원인/결과 중 1~2개를 추가해
   50자 안팎으로 만들 것.
```

Good 예시는 기존 1쌍 유지 + 50~55자 사례 1개 추가.

## 변경 3 — Self-check 명령

출력 형식 직전에 추가:

```
[OUTPUT SELF-CHECK]
출력 직전, summary의 각 줄 글자수(공백·번호 포함)를 세어
45~58자 범위인지 확인할 것.
범위 밖이면 다시 작성한 후 출력할 것.
```

## 변경 4 — 후처리 안전망 (자동 재요청)

엣지 함수 로직에 추가:

1. AI 응답 받은 뒤 `parsed.summary`를 `\n`으로 split
2. 4줄 중 한 줄이라도 `< 40자`면 → 동일 프롬프트로 1회 재호출
   - user 메시지에 `"이전 응답이 너무 짧았음. 각 줄을 반드시 50자 안팎으로 다시 작성"` 추가
3. 재시도해도 짧으면 그대로 반환 (무한루프 방지)
4. 재시도 발생 여부는 `console.log`로 기록

```text
[흐름]
첫 호출 → split 검증 → 짧음 감지 → 재호출(1회) → 반환
                    └─ 충분 → 즉시 반환
```
