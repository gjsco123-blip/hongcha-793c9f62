

# 구문분석 AI 채팅이 멍청해진 원인과 수정 계획

## 진단

### 문제 1: 33개 패턴 과부하
서버 로그에 `Matched 33/55 patterns`가 찍힘. Grammar Track이 모든 문법 태그 패턴을 무조건 포함하므로, DB에 저장된 55개 중 33개가 매번 시스템 프롬프트에 주입됨. 채팅 한 번에 33개 문법 규칙을 읽으니 모델이 혼란에 빠져 엉뚱한 문법으로 답하고, 핵심 지시를 무시함.

### 문제 2: 채팅에서는 "현재 문장의 문법"이 이미 확정되어 있는데, 프롬프트가 이를 반영 안 함
자동생성은 "문장에서 문법을 찾아라"이므로 여러 패턴을 보여주는 게 맞음. 하지만 채팅은 **이미 생성된 구문분석을 수정하는 단계**임. 현재 1번 포인트가 "동격의 접속사 that"이면 관련 패턴은 `명사절` 하나뿐인데, 33개를 전부 주입하니 모델이 다른 패턴을 따라가버림.

### 문제 3: 앵무새 반복
모델이 한번 틀린 답을 하면 대화 히스토리에 그 답이 남고, 다음 요청에서도 같은 히스토리를 보내므로 동일한 오답을 반복함. 시스템 프롬프트에 "사용자가 틀렸다고 지적하면 다시 분석하라"는 지시가 없음.

## 수정 계획

### 1. 채팅용 패턴 주입을 "현재 포인트 태그 전용"으로 축소
- 현재: 33개 전부 주입
- 변경: 현재 수정 대상 포인트의 태그(예: `명사절`)와 매칭되는 패턴 **1~2개만** 주입
- 전체 수정 모드(번호 미선택)에서도 현재 노트에 있는 태그만 추출해서 관련 패턴만 주입

### 2. 시스템 프롬프트에 "현재 분석을 신뢰하라" 지시 추가
- "현재 구문분석 노트의 문법 판단(동격/관계대명사/분사 등)은 선생님이 확인한 것이다. 이 판단을 바꾸지 말고 그대로 유지한 채 표현만 수정하라."
- "사용자가 명시적으로 '이거 관계대명사 아니라 접속사야' 같이 문법 판단 자체를 바꾸라고 하지 않는 한, 기존 문법 분류를 유지하라."

### 3. "사용자 피드백 수용" 지시 추가
- "사용자가 '틀렸어', '아니야', '다시 해줘' 등으로 오류를 지적하면, 이전 답변을 그대로 반복하지 말고 문장을 다시 분석하여 새로운 답변을 제시하라."
- 이 한 줄이 앵무새 반복을 크게 줄임

### 4. `materializePinnedPattern` English swap 로직 단순화
- 현재 English segment 교체가 불안정하게 작동하여 오히려 올바른 AI 출력을 망가뜨리는 경우 있음
- 채팅에서는 후처리 교체를 더 보수적으로: AI 출력이 이미 고정 패턴 태그와 매칭되고 스타일이 유사하면 교체 안 함

## 수정 파일

| 파일 | 변경 |
|------|------|
| `supabase/functions/grammar-chat/index.ts` | 패턴 주입 축소 (현재 태그 전용), 시스템 프롬프트 보강 (현재 분석 신뢰 + 피드백 수용) |

## 기대 결과
- 채팅에 1~2개 관련 패턴만 주입 → 모델 혼란 제거
- 기존 문법 판단을 존중하므로 "동격→주격보어" 같은 엉뚱한 변환 방지
- 사용자가 틀렸다고 하면 다시 분석하므로 앵무새 반복 해소
