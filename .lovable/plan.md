

# WORKBOOK 글자별 개별 inset 적용

## 문제

현재 `CHAR_METRICS`가 글자 종류별(`W`, `O`, `R`, `K`, `B`)로만 정의되어 있어서, 같은 글자가 여러 번 나와도 동일한 `inset`이 적용됨.

하지만 "WORKBOOK"에서:
- `O`가 3번 등장 (인덱스 1, 5, 6) — 각각 상단 직선, 곡선, 우측 직선에 위치
- `K`가 2번 등장 (인덱스 3, 7) — 각각 곡선 진입 전, 우측 직선에 위치

**같은 글자라도 위치(직선 vs 곡선)에 따라 외곽선과의 시각적 거리가 다르게 보이므로**, 글자별이 아니라 **위치별(인덱스별)로 개별 inset을 지정**해야 함.

## 해결

`CHAR_METRICS` 공유 맵 → **8개 글자 각각의 개별 metric 배열**로 변경.

```text
인덱스  글자  구간           현재 inset → 개별 inset
  0     W    상단 직선       0            개별 조정
  1     O    상단 직선       1.0          개별 조정
  2     R    상단~곡선       0            개별 조정
  3     K    곡선            0            기준 (0)
  4     B    곡선~우측       0.8          개별 조정
  5     O    우측 직선       1.0          개별 조정
  6     O    우측 직선       1.0          개별 조정
  7     K    우측 직선       0            개별 조정
```

## 수정 내용

| 파일 | 변경 |
|------|------|
| `src/components/WorkbookPdfDocument.tsx` | `CHAR_METRICS` Record를 배열 `LETTER_METRICS[8]`로 교체. 각 인덱스마다 `{ char, w, inset }`을 독립적으로 지정. `getArcLetters()`에서 `m = LETTER_METRICS[i]`로 참조. |

이렇게 하면 같은 `O`라도 상단에 있는 것과 우측에 있는 것의 inset을 따로 조정할 수 있어서, 모든 글자의 외곽선 거리를 개별적으로 맞출 수 있음.
