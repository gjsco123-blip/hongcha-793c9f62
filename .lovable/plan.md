

# MEMO 줄 수 과소 계산 수정

## 문제
`estimateMemoLines` 함수가 콘텐츠 높이를 실제보다 크게 추정하여, 남은 공간이 적다고 잘못 판단하고 MEMO를 4줄만 생성함. 실제로는 10줄이 충분히 들어갈 공간이 있음.

## 원인 분석

| 항목 | 현재 값 | 실제 적정 값 | 오차 방향 |
|------|---------|-------------|-----------|
| 영어 문장 줄바꿈 | 60자/줄 | 75자/줄 | 과대추정 |
| 스스로 분석 줄바꿈 | 65자/줄 | 72자/줄 | 과대추정 |
| 안전 마진 | 3줄 (54pt) | 1줄 (18pt) | 과도한 여유 |

7문장 기준 영어만으로도: 각 문장 평균 100자라면 현재 `ceil(100/60)=2줄`, 실제 `ceil(100/75)=2줄`로 짧은 문장에선 차이가 적지만, 슬래시(`/`)가 포함된 청크 텍스트는 더 길어져서 차이가 커짐.

## 수정 내용

### 파일: `src/components/PdfDocument.tsx` - `estimateMemoLines` 함수

1. **영어 문장 줄바꿈 추정 완화**: `/60` -> `/75`
   - 가용 너비 418pt, fontSize 9pt SemiBold 기준 약 75자가 적정

2. **스스로 분석 줄바꿈 추정 완화**: `/65` -> `/72`
   - 패딩 제외 가용 너비 390pt, fontSize 9pt Regular 기준 약 72자가 적정

3. **안전 마진 축소**: `safetyMargin = 3` -> `safetyMargin = 1`
   - 개별 항목 추정이 이미 약간 보수적이므로, 전체 안전 마진은 1줄(18pt)이면 충분
   - 최대 10줄 제한 자체가 안전장치 역할을 함

이 세 가지 조정으로, 공간이 충분할 때 최대 10줄까지 MEMO가 생성되며, 여전히 2페이지를 초과하지 않음.
