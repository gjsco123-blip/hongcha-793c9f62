

# 고정 패턴 프롬프트 톤 강화: '참고용' → '필수 규칙'

## 변경 대상

`supabase/functions/grammar/index.ts` — lines 831-838

## 현재 프롬프트 (약한 톤)

```
아래는 현재 문장과 관련 있는 고정 스타일 패턴이다.
해당 문법 요소가 문장에 실제로 존재할 때, 이 형식과 설명 스타일을 따르라.
```

"따르라"라고 되어 있지만 전체 맥락이 "참고" 느낌이라 모델이 자기 방식을 우선함.

## 변경 후 프롬프트 (강제 톤)

```
[필수 적용 규칙 — 고정 패턴]
아래 패턴은 사용자가 직접 지정한 필수 설명 형식이다.
해당 문법 요소가 문장에 존재하면, 반드시 아래 형식과 설명 스타일을 그대로 따라야 한다.
너 자신의 설명 방식이나 표현을 사용하지 말고, 아래 패턴의 구조·어휘·종결 방식을 정확히 복제하라.
___가 있으면 해당 문장의 실제 단어로 교체하라.
문장에 해당 문법 요소가 없으면 이 패턴을 완전히 무시하라. 억지로 적용하지 말 것.
```

## 핵심 차이점

1. 헤더를 `[필수 적용 규칙]`으로 변경 — 모델이 우선순위를 높게 인식
2. "사용자가 직접 지정한" 표현 추가 — 모델이 사용자 의도를 존중하도록 유도
3. "너 자신의 방식을 사용하지 말고 정확히 복제하라" — 모델의 자체 판단 억제
4. 관련성 필터(Layer 1)는 그대로 유지 — 무관한 패턴 유입 방지는 변함없음
