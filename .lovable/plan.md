
체크 결과

- 네 눈이 맞아. 지금은 `BOOK` 쪽이 `WORK`의 `K`보다 외곽선에서 조금 더 멀어 보여.
- 특히 `B`, 첫 번째 `O`, 세로로 내려가는 `O/O/K` 구간이 더 떠 보이는 편이야.
- 그래서 기준 간격은 네가 말한 것처럼 `WORK`의 `K`와 외곽선 사이 거리로 잡는 게 맞아.

왜 이렇게 보이냐면

- 현재 `getArcLetters()`는 모든 철자에 같은 크기값(`charW = 4.5`, `charH = 6.5`)을 써서 중심 보정을 하고 있어.
- 그런데 `W`, `O`, `B`, `K`는 실제 폭도 다르고, 검은 획이 차지하는 시각적 중심도 달라.
- 즉 지금 로직은 “글자 박스 중심”은 맞추지만, 사람이 보는 건 “검은 글자와 외곽선 사이의 거리”라서 `BOOK`가 더 멀게 읽혀.
- 특히 둥근 글자(`B/O`)는 같은 좌표에 있어도 더 바깥으로 뜬 것처럼 보이기 쉬워.

왜 한 번에 안 맞았는지

- 이건 단순 좌표 문제가 아니라 optical alignment 문제라서 그래.
- `@react-pdf/renderer`는 text-on-path를 직접 지원하지 않고, 각 철자를 절대배치 + 회전으로 흉내 내고 있어서 수학적으로 맞아도 눈에는 다르게 보일 수 있어.
- 어려운 작업은 맞지만, 고칠 수 있는 문제야. 방향도 지금은 꽤 명확해.

구현 계획

1. 기준 간격 고정
   - `WORK`의 `K`와 외곽선 사이 간격을 기준값으로 삼음.

2. 공통 보정값 제거
   - 지금처럼 모든 철자에 동일한 `charW/charH`를 쓰지 않고, 철자별 폭/시각 중심 보정값으로 바꿈.

3. 경로는 유지, 거리 계산만 정교화
   - 지금의 `상단 직선 → 모서리 곡선 → 우측 직선` 경로는 유지.
   - 다만 각 철자를 경로 위에 놓은 뒤, 그 지점의 법선 방향으로 동일한 “보이는 거리”만큼 배치되도록 수정.

4. 둥근 글자 optical correction 추가
   - `B/O/O`처럼 둥근 글자는 현재보다 안쪽으로 조금 더 당겨서 `K`와 같은 간격으로 보이게 함.

5. 간격과 외곽선 거리 분리
   - 철자 사이 spacing 문제와 외곽선에서 뜨는 문제를 따로 다룸.
   - 지금은 둘이 섞여 있어서 하나를 맞추면 다른 하나가 어색해지는 상태야.

기술 메모

- 수정 파일: `src/components/WorkbookPdfDocument.tsx`
- 수정 대상: `getArcLetters()`
- 현재 핵심 원인:
  - 고정값 `charW = 4.5`
  - 고정값 `charH = 6.5`
  - 철자별 차이를 반영하지 않는 공통 중심 보정
- 변경 방향:
  - 철자별 metric map
  - segment/top-arc-right별 normal 기준 배치
  - `BOOK`만 따로 억지로 맞추는 게 아니라, 전체 단어가 같은 간격으로 보이도록 재정렬

완성 기준

- `WORK`의 `K` gap이 기준
- `B/O/O/K`도 같은 gap로 보여야 함
- 곡선 구간과 세로 구간 연결이 자연스러워야 함
- 글자 간격이 넓어져도 외곽선에서 뜬 느낌이 없어야 함
