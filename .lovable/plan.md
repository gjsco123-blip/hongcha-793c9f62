
핵심 답변

- 아니, 지금은 수치적으로도 같지 않아.
- 현재 코드 기준 실제로 쓰는 offset은 `W 4.45 / O 4.10 / R 4.60 / K 4.60 / B 4.20 / O 4.10 / O 4.10 / K 4.60`이라서 애초에 동일값이 아님.
- 더 중요한 건 이 수치가 네가 원하는 "외곽선 → 검은 글자 외곽" 거리가 아니라, "경계선 기준점 → 추정 글자 중심" 거리라는 점이야. 그래서 숫자를 비슷하게 맞춰도 눈에는 안 맞아.

왜 BOOK이 더 떨어져 보이냐

1. `O`, `B`는 둥근 글자라 검은 외곽이 박스 끝까지 꽉 차지 않아.
   - 같은 중심거리라도 `R`, `K`보다 더 안쪽에 있어 보여.
2. `B/O/O/K`는 곡선~우측 세로 구간이라 회전이 들어감.
   - 지금 로직은 `w`와 고정 `charH`로 사각 박스 중심만 보정해서, 회전된 실제 글자 외곽선 위치를 못 맞춤.
3. 지금 `inset` 하나로
   - 외곽선 gap
   - 글자 anchor 보정
   - 약간의 spacing 보정
   을 같이 해결하려 해서, 한 글자 맞추면 다른 글자가 틀어지는 구조야.

결론

- 방법은 있어.
- 그런데 지금 방식처럼 Text 8개를 "대충 width + inset"으로 놓는 방식은 구조적으로 한계가 있어서 계속 안 맞는 거야.
- 네가 원하는 건 "시각적 외곽선 gap 통일"이고, 이건 `중심점` 기준이 아니라 `글자 실제 외곽선` 기준으로 풀어야 해.

권장 해결책

1. 기준점을 `R`로 고정
   - 목표값을 `R`의 실제 외곽선 gap으로 잡음.
2. 각 글자마다 `inset`만 두지 말고, 최소 3개 값을 분리
   - `advanceAdjust`: 경로 따라 앞뒤 위치
   - `normalGapAdjust`: 외곽선에서 떨어지는 거리
   - `anchorBias`: 회전/앵커 오차 보정
3. "박스 중심"이 아니라 "글자 외곽선의 border 방향 돌출량" 기준으로 계산
   - 목표식:
   - `border -> glyph outline distance = constant`
   - 지금은 사실상
   - `border -> estimated glyph center distance ≈ constant`
   - 이걸 하고 있어서 오차가 남아.
4. `B/O/O/K`는 같은 글자라도 전부 개별값 유지
   - 위치와 회전각이 달라서 절대 묶으면 안 됨.

내가 보기엔 가장 안정적인 방법

- 가장 정확: `WORKBOOK`을 텍스트가 아니라 벡터 아웃라인/SVG path로 고정
  - 장점: 한번 맞추면 절대 안 흔들림
  - 단점: 문구를 자주 바꾸기엔 불편
- 그다음으로 현실적: 지금 Text 방식 유지하되, `LETTER_METRICS`를 `w + inset` 수준이 아니라 "외곽선 기준 보정값" 체계로 바꾸기
  - 이게 현재 구조를 살리면서 맞출 수 있는 방법

내 판단

- 지금 상태는 "수치적으로도 동일하지 않고, 수치 정의 자체도 네가 원하는 거리 정의가 아님".
- 그래서 계속 안 맞는 게 정상이고, 네 눈이 맞아.
- 진짜 동일하게 보이게 하려면 `BOOK만 inset 조정`이 아니라, 배치 모델 자체를 `글자 외곽선 기준`으로 바꿔야 해.

구현 계획

1. `src/components/WorkbookPdfDocument.tsx`의 `getArcLetters()` 배치 기준을 재정의
   - `baseOffset - inset` 중심 배치에서 벗어남
2. `LETTER_METRICS`를 단순 `{ w, inset }`에서 `{ advanceAdjust, normalAdjust, anchorX, anchorY }` 수준으로 분리
3. `R`을 기준 gap으로 고정하고 각 철자의 실제 시각 gap을 그 기준에 맞게 정렬
4. 특히 `K(곡선)`, `B`, `O(우측 2개)`, `K(우측)`를 별도 보정
5. 필요하면 최종적으로는 `WORKBOOK` 라벨만 SVG/outline 방식으로 고정하는 fallback 준비

기술 메모

- 현재 파일: `src/components/WorkbookPdfDocument.tsx`
- 현재 문제 핵심:
  - `LETTER_METRICS`가 실제 외곽선이 아닌 근사 폭(`w`)만 사용
  - `charH = 6.5` 고정
  - 회전 후 실제 glyph silhouette가 아니라 사각형 기준 anchor 보정
- 현재 effective offset:
  - W 4.45
  - O 4.10
  - R 4.60

  - B 4.20
  - O 4.10
  - O 4.10
  - K 4.60

완성 기준

- `W/O/R/K/B/O/O/K` 전부가 "검은 글자 외곽" 기준으로 같은 gap처럼 보여야 함
- `BOOK`가 더 떠 보이면 실패
- `R` 기준 gap이 곡선/세로 구간까지 그대로 이어져야 함
- spacing과 gap을 분리해서, 간격 맞춘다고 외곽선 distance가 흔들리지 않아야 함
